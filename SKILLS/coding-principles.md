# Coding Principles

## Main Rules

- Write readable code
- Keep functions small
- Use clear names
- Avoid duplicated logic
- Separate responsibility
- Prefer simple solutions

## SOLID Summary

- S: One class should have one responsibility
- O: Open for extension, closed for modification
- L: Subclasses should work as parent classes
- I: Avoid large interfaces
- D: Depend on abstraction, not concrete classes
